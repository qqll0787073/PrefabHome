# PH-008A Contract Foundation

PH-008A adds a contract foundation for confirmed purchase orders only. It does not implement electronic signatures, PDF generation, DocuSign, Adobe Sign, payments, invoices, shipping, customs, production milestones, notifications, or workflow automation.

## Database Design

Migration: `supabase/migrations/0016_contract_foundation.sql`

Tables:

- `public.contracts`
- `public.contract_events`

`contracts.contract_number` is generated by the database as `CON-YYYY-NNNNNN` from `public.contract_number_seq`. Application code never supplies contract numbers.

Contract statuses:

- `draft`
- `ready`

Lifecycle:

- A Buyer may create one draft contract from their own confirmed purchase order.
- A Buyer may update title, governing law, and terms while the contract is draft.
- A Buyer may mark a draft contract ready.
- Ready contracts are immutable in PH-008A.
- Manufacturer and Admin contract surfaces are read-only in PH-008A.

## Snapshot Model

Each contract stores immutable JSON snapshots captured at creation time:

- `purchase_order_snapshot`
- `buyer_snapshot`
- `manufacturer_snapshot`
- `quote_snapshot`
- `product_snapshot`
- `line_items_snapshot`

Snapshots preserve the accepted commercial context even if the source purchase order, quote, product, buyer profile, or manufacturer application changes later.

## Trusted RPCs

Public authenticated RPCs:

- `create_contract_from_po(po_uuid uuid)`
- `update_contract_draft(contract_uuid uuid, contract_title_text text, governing_law_text text, contract_terms_text text)`
- `mark_contract_ready(contract_uuid uuid)`

Internal-only helpers and trigger functions explicitly revoke execution from `PUBLIC`, `anon`, and `authenticated`.

## RLS Summary

Anonymous users have no contract table access.

Authenticated read access:

- Buyer can read contracts where `buyer_id = auth.uid()`.
- Owning Manufacturer can read contracts assigned to their manufacturer profile.
- Admin can read all contracts.

Writes:

- Direct table writes are revoked and blocked by triggers.
- Contract writes occur only through the trusted RPCs.
- Contract events are generated by trusted database functions.
- Admin and Manufacturer cannot mutate contracts in PH-008A.

## Frontend

Buyer Portal:

- Lists contracts visible to the Buyer.
- Creates a contract from eligible confirmed purchase orders.
- Edits draft title, governing law, and terms.
- Marks a draft contract ready.

Manufacturer Portal:

- Lists assigned contracts and events.
- Read-only.

Admin Portal:

- Lists all contracts, events, and source snapshot identifiers.
- Read-only.

## Verification

Rollback-only SQL verification is in:

- `supabase/tests/contract_security.sql`

It verifies creation from confirmed purchase orders, duplicate prevention, immutable snapshots, role-based reads, anonymous denial, trusted events, and blocked direct writes.
