# PH-006B Quote Builder

PH-006B adds manufacturer-authored RFQ quotes without adding buyer review actions. Acceptance, rejection, revision requests, purchase orders, payments, shipping, invoices, notifications, and PH-006C workflows remain deferred.

## Lifecycle

Quote statuses are:

- `draft`
- `submitted`
- `superseded`
- `expired`
- `withdrawn`

Manufacturers create and edit only their own draft quotes for RFQs assigned to their manufacturer profile. Submitted, superseded, expired, and withdrawn quotes are immutable through normal client paths.

## Version Model

Quote versions are assigned by the database per RFQ using `version = max(version) + 1`.

Clients cannot choose:

- `manufacturer_id`
- `created_by`
- `version`
- `subtotal`
- `submitted_at`

`create_rfq_quote_revision(quote_uuid)` copies a submitted or superseded quote into a new draft with the next version number and copies its line items. Creating a revision does not move the RFQ status.

## Line Items

Quote line items live in `public.rfq_quote_items`.

Supported item types are:

- `product`
- `customization`
- `packaging`
- `freight`
- `insurance`
- `tax`
- `discount`
- `other`

`amount` is generated by PostgreSQL as `quantity * unit_price`. `subtotal` is recalculated by database triggers from line item amounts.

Discount lines are modeled as non-negative line items for PH-006B. Negative pricing is not supported yet.

## Trusted Submission

Manufacturers use `submit_rfq_quote(quote_uuid)` to submit a quote. The RPC runs atomically and verifies:

- the caller is authenticated
- the caller owns the RFQ manufacturer profile
- the quote is a draft
- the RFQ is in `manufacturer_review`
- at least one line item exists

Submission then:

- recalculates subtotal
- marks previous submitted quotes for the RFQ as `superseded`
- sets the draft quote to `submitted`
- sets `submitted_at`
- moves the RFQ from `manufacturer_review` to `quoted`
- creates a trusted `quote_created` RFQ event

The UI must not insert `quote_created` events directly.

## RFQ Integration

PH-006B uses the PH-006A RFQ state machine. Quote submission is the only new trusted path for `manufacturer_review -> quoted`.

If a manufacturer creates a draft from a newly submitted RFQ, `create_rfq_quote_draft(rfq_uuid)` moves the RFQ to `manufacturer_review` using the existing manufacturer review transition. Submitting the quote then moves it to `quoted`.

## RLS Model

Anonymous users have no quote access.

Manufacturers can:

- read quotes for RFQs assigned to their manufacturer profile
- create quote drafts only through trusted RPCs
- edit/delete only their own draft quotes
- add/update/delete line items only on their own draft quotes

Buyers can:

- read non-draft quotes for their own RFQs
- read associated line items
- never insert, update, or delete quote rows/items

Admins can:

- read all quotes and line items
- view quotes in the Admin RFQ Management UI
- not mutate quote lifecycle in PH-006B UI

## Service Layer

`src/lib/quotes.ts` owns quote data access and validation helpers.

Lifecycle operations use RPCs:

- `createQuoteDraft()`
- `submitQuote()`
- `createQuoteRevision()`
- `deleteQuoteDraft()`

Draft field and line item edits use RLS-protected table writes.

## UI Surfaces

Manufacturer Portal:

- Quote Builder from RFQ Inbox
- draft metadata form
- dynamic line items
- subtotal display
- save draft
- submit quote
- delete draft
- create revision from submitted/superseded quotes

Buyer Portal:

- Quote Received indicator in My RFQs
- read-only quote versions and line items
- no accept/reject/request-revision actions

Admin Portal:

- read-only quote detail under RFQ Management
- no quote mutation controls

## Deferred To PH-006C

- buyer quote acceptance
- buyer rejection
- buyer revision requests
- quote negotiation state
- purchase order creation
- payment or invoice workflows
- shipping workflows
- notifications
