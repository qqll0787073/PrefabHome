## Migration Review

Confirm:

- Migration `0011` is additive.
- Migrations `0001` through `0010` are unchanged.
- Tables exist after local/rollback migration setup:
  - `public.rfqs`
  - `public.rfq_messages`
  - `public.rfq_events`
- RLS is enabled on all new tables.
- Anonymous access is not granted.
- `product_snapshot` is JSONB, generated by the database, and immutable after insert.
- Message `sender_role` is database-derived, not client-trusted.
- RFQ events are constrained to the documented event-type list.
- Authenticated users do not have direct `INSERT` or `UPDATE` grants on `public.rfq_events`.
- Lifecycle events are generated by RFQ triggers.
- Manufacturer reply events are generated by the RFQ message trigger.
- Opened events are generated by role-derived RPCs.
- `quote_created` is deferred until PH-006B and cannot be emitted by ordinary clients.
- `requested_quantity` is positive numeric quantity.
- `requested_currency` is normalized and constrained to a three-letter uppercase code.
- `incoterm` is nullable and constrained to supported values.

## Lifecycle Matrix

Allowed transitions:

- `draft -> submitted`
- `draft -> cancelled`
- `submitted -> manufacturer_review`
- `submitted -> cancelled`
- `submitted -> expired`
- `manufacturer_review -> quoted`
- `manufacturer_review -> expired`
- `quoted -> buyer_review`
- `quoted -> expired`
- `buyer_review -> accepted`
- `buyer_review -> declined`
- `buyer_review -> expired`

Blocked transitions to verify:

- `draft -> accepted`
- `draft -> quoted`
- `submitted -> accepted`
- `accepted -> draft`
- `declined -> quoted`
